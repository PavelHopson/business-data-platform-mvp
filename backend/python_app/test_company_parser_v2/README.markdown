# 📊 Сбор данных о российских компаниях по ИНН

## Описание проекта

Проект представляет собой модульный парсер для сбора, агрегации и унификации ключевой бизнес-информации о российских компаниях по их ИНН. 
Результаты парсинга из нескольких источников объединяются в единую, стандартизированную JSON-модель.

## Источники данных

Для обеспечения полноты и верификации данных используются три независимых источника:

* **pb.nalog.ru (ФНС):** Базовые данные, официальное название, статус, основной ОКВЭД (используется как основной, верифицирующий источник).
* **list-org.com:** Финансовые показатели (выручка, прибыль), руководители, учредители, контакты, судебные дела.
* **rusprofile.ru:** Регистрационные данные (ОГРН, дата), руководители, учредители, судебные дела.

## Особенности

* **Агрегация данных:** Объединение информации из трех источников с приоритетом на базовые данные ФНС.
* **Унификация:** Все собранные данные приводятся к единой структуре (`models.py`).
* **Устойчивость к блокировкам:** Использование ротации User-Agent, случайных задержек и продвинутой маскировки Selenium (для ФНС).
* **Модульность:** Архитектура на основе абстрактного класса `BaseParser` позволяет легко добавлять новые источники.
* **Детальное логирование:** Весь процесс фиксируется в консоли и в файле `parser.log`.

## Установка

**1. Клонировать репозиторий:**

Установить зависимости:

Bash

pip install -r requirements.txt

Запуск

Парсер поддерживает три режима запуска из командной строки:

1. Парсинг одного ИНН (результат выводится в консоль и сохраняется в файл):

Bash

python main.py --inn 7707083893

2. Парсинг нескольких ИНН (список через запятую):

Bash

python main.py --inn 7707083893,7736207543,9703073496

3. Парсинг из файла (ИНН должны быть в текстовом файле, по одному на строку):

Bash

python main.py --file inns.txt

Для всех режимов можно указать свой путь к выходному файлу:

Bash

python main.py --file inns.txt --output my_results.json

Результаты
После выполнения парсинга в директории output/final_JSON/ (или по указанному вами пути) будет создан файл output.json (по умолчанию), содержащий массив JSON-объектов успешно спарсенных компаний. Промежуточные результаты (отчеты ФНС, JSON-файлы с List-Org и RusProfile) сохраняются в соответствующих поддиректориях в output/.

## Структура проекта

company_parser/
├── main.py                 # Точка входа для запуска парсера
├── config.py               # Настройки: пути, User-Agent'ы, задержки
├── models.py               # Модели данных (CompanyData, ParsingResult)
├── company_parser.py       # Основной класс-оркестратор парсеров
├── process_company.py      # Функция для объединения JSON-данных (альтернатива company_parser)
├── parsers/                # Парсеры для источников
│   ├── __init__.py
│   ├── base_parser.py      # Базовый класс парсера
│   ├── pb_nalog_parser.py  # Парсер для pb.nalog.ru (Selenium)
│   ├── listorg_parser.py   # Парсер для list-org.com
│   └── rusprofile_parser.py # Парсер для rusprofile.ru
├── requirements.txt        # Зависимости
├── parser.log              # Лог выполнения
├── README.markdown         # Документация
└── output/                 # Директория для результатов
    ├── pb_nalog_table/     # XLSX-файлы от pb.nalog.ru
    ├── pb_nalog_JSON/      # JSON из pb.nalog.ru
    ├── listorg_JSON/       # JSON из list-org.com
    ├── rusprofile_JSON/    # JSON из rusprofile.ru
    └── final_JSON/         # Финальные JSON-файлы
